## 发展
VCS（version control system）出现以前
1. 目录拷贝区分不同版本
2. 公共文件容易被覆盖
3. 成员沟通成本很高，代码集成效率低下

集中式 VCS
1. 集中的版本管理服务器
2. 具备文件版本管理和分支管理能力
3. 客户端必须时刻和服务器连接

分布式 VCS
1. 服务器和客户端均有完整的版本库
2. 脱离服务端，客户端照样可以管理版本
3. 查看历史和版本比较等多数操作，都不需要访问服务器，比集中式更能提高版本管理效率
